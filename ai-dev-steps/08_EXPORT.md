# 阶段 8：导出功能

> ✅ 已完成（2026-06-18）

## 本阶段目标

支持将查询结果导出为 JSON / NDJSON / CSV。

本阶段只实现导出，不实现导入。

## 给 AI 的提示词

```text
请阅读 ES_DESKTOP_CLIENT_PLAN.md。

本次只实现【阶段 8：导出功能】。

目标：
支持将查询结果导出为 JSON / NDJSON / CSV。

要求：
1. 支持导出当前查询结果。
2. 支持 JSON 格式导出。
3. 支持 NDJSON Bulk API 兼容格式导出。
4. 支持 CSV 导出。
5. CSV 只处理 _source 第一层字段。
6. 嵌套对象和数组转换为 JSON 字符串。
7. 使用系统保存文件弹窗。
8. 导出完成后提示保存路径。
9. 不要实现导入功能。

请先说明三种导出格式的转换规则，再开始实现。
```

## 推荐 IPC 接口

```text
export:execute
```

## JSON 导出格式

```json
[
  {
    "_id": "1",
    "_source": {
      "name": "张三",
      "age": 18
    }
  },
  {
    "_id": "2",
    "_source": {
      "name": "李四",
      "age": 20
    }
  }
]
```

## NDJSON 导出格式

```json
{"index":{"_index":"users","_id":"1"}}
{"name":"张三","age":18}
{"index":{"_index":"users","_id":"2"}}
{"name":"李四","age":20}
```

## CSV 导出规则

```text
1. 只导出 _source 第一层字段。
2. 第一行是表头。
3. 普通值直接输出。
4. 对象转 JSON 字符串。
5. 数组转 JSON 字符串。
6. 需要处理中文编码，避免乱码。
```

## MVP 限制

```text
1. 默认导出当前查询结果。
2. 最大导出数量可先限制为 10000 条。
3. 暂不实现 Scroll 大批量导出。
```

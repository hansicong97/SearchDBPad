# V0.2.2 发布说明 · UI 布局优化与连接目录

## 背景

本方案针对当前界面 4 个体验问题：

1. 左侧连接列表过多时需要稳定出现滚动条。
2. 连接列表支持新建目录，用目录分类保存连接。
3. 右侧详情区域不需要外边距，内容直接铺满可用区域。
4. 右侧索引列表、文档列表过长时需要在列表内部滚动，不撑开整个页面。

当前代码已经有一部分滚动布局基础，但还不完整：

- `src/renderer/App.tsx` 的 `Content` 有 `padding: 16`，导致右侧详情区域四周留白。
- `src/renderer/pages/WorkspacePage.tsx` 的外层 `Card` 与 body 都有 padding / gap，使详情界面无法铺满。
- `src/renderer/components/AppSidebar.tsx` 的连接列表容器已有 `overflowY: 'auto'`，但父容器缺少 `minHeight: 0`，在 flex 布局中可能导致滚动区域无法正确收缩。
- `src/renderer/components/IndexList.tsx` 的外层有 `overflow: hidden` 和列表容器 `overflow: auto`，但 AntD Table 未设置纵向滚动高度，数据量大时分页和表格可能一起撑高。
- `src/renderer/components/DocumentPanel.tsx` 当前 Table 未设置 `scroll.y`，文档行内容多时容易让 Tab 内容撑开。
- `src/shared/ipc.ts` 的 `EsConnection` 当前没有目录字段，`src/main/store/connectionStore.ts` 也只保存 `connections`，因此目录功能需要数据结构和 IPC 同步扩展。

---

## 总体目标

- 不引入新依赖。
- 不改变 Electron 安全基线：`contextIsolation: true`、`nodeIntegration: false` 保持不变。
- 目录数据仍通过主进程 `electron-store` 持久化，renderer 不直接访问文件系统。
- 旧版本已有连接自动落到“未分组”，保证向后兼容。
- 滚动只发生在列表区域，不让整个 `body` 或右侧页面滚动。

## 方案一：左侧连接列表滚动条

### 目标

当连接数量超过左侧可视高度时，只滚动连接列表区域；顶部“连接 / 共 N”与“新建连接”按钮固定不动。

### 修改文件

- `src/renderer/components/AppSidebar.tsx`
- 必要时微调 `src/renderer/App.tsx`

### 修改点

#### 1. `AppSidebar` 根容器补充 `minHeight: 0`

当前：

```tsx
<div
  style={{
    display: 'flex',
    flexDirection: 'column',
    height: '100%',
    padding: 12
  }}
>
```

建议：

```tsx
<div
  style={{
    display: 'flex',
    flexDirection: 'column',
    height: '100%',
    minHeight: 0,
    padding: 12
  }}
>
```

#### 2. 连接列表容器补充 `minHeight: 0`

当前：

```tsx
<div style={{ flex: 1, overflowY: 'auto', marginRight: -8, paddingRight: 8 }}>
```

建议：

```tsx
<div
  style={{
    flex: '1 1 auto',
    minHeight: 0,
    overflowY: 'auto',
    overflowX: 'hidden',
    marginRight: -8,
    paddingRight: 8
  }}
>
```

### 说明

在 flex 子项中，仅设置 `flex: 1` 不一定能让元素收缩到父容器高度内；`minHeight: 0` 是让滚动容器生效的关键。

## 方案二：连接目录 / 分组保存连接

### 推荐交互

左侧连接区改为：

```text
连接                         [+目录]
[新建连接]

未分组
  本地 ES
  测试集群

生产环境
  prod-es-01
  prod-es-02

客户项目 A
  customer-a-es
```

目录支持：

- 新建目录
- 重命名目录
- 删除目录
- 连接归属目录
- “未分组”为系统默认分组，不允许删除、不需要持久化为目录记录

第一版推荐采用“编辑连接时选择目录”的方式移动连接，避免一次性加入拖拽排序、右键菜单等复杂交互。

扩展 `EsConnection`：


### IPC 设计

修改 `src/shared/ipc.ts` 的 `IpcChannels`：

```ts

ConnectionFolderCreate: 'connection-folder:create',

```

```ts

```

`src/preload/index.ts` 暴露：

```ts
connectionFolders: {
  list: () => Promise<ApiResponse<ConnectionFolder[]>>
  create: (input: ConnectionFolderInput) => Promise<ApiResponse<ConnectionFolder>>
  update: (input: ConnectionFolderInput) => Promise<ApiResponse<ConnectionFolder>>
  delete: (id: string) => Promise<ApiResponse<{ id: string }>>
}
```

### 持久化层

修改 `src/main/store/connectionStore.ts`。

当前 schema：

```ts

  connections: EsConnection[]

```

建议改为：

```ts
interface ConnectionStoreSchema {
  connections: EsConnection[]
```

默认值：

```ts

```

新增函数：

```ts

```

### 服务层

修改 `src/main/services/connection.service.ts`。

新增目录 CRUD：

- `listConnectionFolders()`
- `createConnectionFolder(input)`
- `updateConnectionFolder(input)`
- `deleteConnectionFolder(id)`

规则：

1. 目录名必填，trim 后不能为空。
2. 同名目录不允许重复（大小写不敏感）。
3. 删除目录时不删除连接，只把该目录下连接的 `folderId` 置为 `null`。
4. 创建 / 更新连接时校验 `folderId` 是否存在；不存在则返回错误。
5. 旧连接没有 `folderId` 时视为 `null`，显示在“未分组”。

## 方案四：索引列表、文档列表内部滚动

### 目标

- 索引列表过长时，只滚动表格主体；顶部搜索 / 新建 / 刷新栏固定。
- 文档列表过长时，只滚动文档表格主体；顶部按钮栏固定。
- 不让整个详情页或 body 滚动。

### 4.1 IndexList.tsx

当前结构已经接近目标：

```tsx
<div root height=100 overflow=hidden>
  <div toolbar flex=0 />
  <div tableWrapper flex=1 overflow=auto>
    <Table scroll={{ x: 'max-content' }} />
  </div>
</div>
```

建议强化：

```tsx
<div style={{ flex: '1 1 auto', minHeight: 0, overflow: 'hidden' }}>
  <Table<EsIndexInfo>
    ...
    scroll={{ x: 'max-content', y: 'calc(100vh - 260px)' }}
    pagination={{ ... }}
  />
</div>
```

更稳妥的第一版做法：

- 外层 `tableWrapper` 使用 `overflow: 'auto'`。
- Table 保留 `scroll.x`。
- 若发现 AntD Table 仍撑高，再引入一个小 hook 测量 wrapper 高度后传给 `scroll.y`。

推荐实现顺序：

1. 先补齐 flex 链路：`minHeight: 0` + `overflow: hidden/auto`。
2. 手测 100+ 索引。
3. 若仍无纵向滚动，再增加 `useElementSize` 计算 `scroll.y`。

### 4.2 DocumentPanel.tsx

当前：

```tsx
<div style={{ display: 'flex', flexDirection: 'column', height: '100%' }}>
  <Space toolbar />
  <Table ... />
</div>
```

建议改为：

```tsx
<div
  style={{
    display: 'flex',
    flexDirection: 'column',
    height: '100%',
    minHeight: 0,
    overflow: 'hidden'
  }}
>
  <Space style={{ marginBottom: 12, flex: '0 0 auto' }} size="middle">
    ...
  </Space>

  <div style={{ flex: '1 1 auto', minHeight: 0, overflow: 'auto' }}>
    <Table<DocumentHit>
      ...
      scroll={{ x: 'max-content' }}
    />
  </div>
</div>
```

如果需要固定表头，再升级为：

```tsx
scroll={{ x: 'max-content', y: tableHeight }}
```

### 4.3 IndexDetailPanel.tsx / Tabs 容器

AntD Tabs 在 flex 中常见问题是 `.ant-tabs-content-holder` 默认不一定正确收缩。当前代码只给 `Tabs` 本身设置了 flex：

```tsx
<Tabs
  style={{ flex: 1, minHeight: 0, display: 'flex', flexDirection: 'column' }}
/>
```

建议增加一个局部 className 并在 CSS 中约束内部结构。

新增 class：

```tsx
<Tabs className="workspace-tabs" ... />
```

在 `src/renderer/styles/global.css` 增加：

```css
.workspace-tabs,
.workspace-tabs > .ant-tabs-content-holder,
.workspace-tabs > .ant-tabs-content-holder > .ant-tabs-content,
.workspace-tabs > .ant-tabs-content-holder > .ant-tabs-content > .ant-tabs-tabpane {
  min-height: 0;
  height: 100%;
}

.workspace-tabs > .ant-tabs-content-holder {
  flex: 1 1 auto;
  overflow: hidden;
}
```

这样每个 tab 的 children 容器才能在剩余高度内滚动。

## 风险与注意事项

1. **目录功能是数据模型变更**：需要 main / preload / shared / renderer 全链路同步，不能只改前端显示。
2. **旧数据兼容**：`folderId` 必须是可选字段，旧连接自动进入未分组。
3. **删除目录不要删除连接**：推荐移动到未分组，避免误删连接配置。
4. **滚动高度不要写死太多**：优先用 flex + `minHeight: 0`；只有 AntD Table 仍撑开时再测量容器高度传 `scroll.y`。
5. **右侧去 margin 后要保留必要内部 padding**：例如集群信息卡、工具栏仍需要少量内边距，否则文字贴边太紧。
6. **Tabs 滚动链路容易断**：需要同时约束 `Tabs`、`.ant-tabs-content-holder`、`.ant-tabs-content`、`.ant-tabs-tabpane` 的 `min-height` / `height`。

---

## 建议本次实际实施范围

建议拆成两个提交/阶段：

1. **UI 布局阶段**：滚动条 + 右侧铺满。风险低，可快速验证。
2. **连接目录阶段**：数据模型 + IPC + UI 分组。风险较高，单独验证持久化和旧数据兼容。

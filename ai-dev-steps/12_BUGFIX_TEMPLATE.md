# Bug 修复沟通模板

当开发过程中出现错误时，不要只告诉 AI “报错了”。要提供完整上下文。

## 给 AI 的提示词

```text
我在执行【具体操作】时遇到错误。

错误信息如下：

```text
把完整错误粘贴在这里
```

当前操作步骤：
1. 我先……
2. 然后……
3. 最后……

当前期望结果：
```text
描述你本来希望看到什么
```

实际结果：
```text
描述实际发生了什么
```

请你：
1. 先判断错误原因。
2. 不要直接大范围重写。
3. 只修改和这个错误相关的代码。
4. 不要实现新功能。
5. 修改后告诉我如何验证。
```

## 如果是启动错误

```text
我在启动应用时遇到错误。

执行的命令：
```text
npm run dev
```

错误信息：
```text
粘贴完整错误
```

请只修复启动相关问题，不要改业务功能。
```

## 如果是 TypeScript 错误

```text
我在执行 TypeScript 检查或构建时遇到错误。

执行的命令：
```text
npm run build
```

错误信息：
```text
粘贴完整错误
```

请逐条分析错误原因，只修改类型或相关代码，不要重构无关模块。
```

## 如果是 Elasticsearch 连接错误

```text
我在测试 Elasticsearch 连接时失败。

连接配置：
```text
地址：http://localhost:9200
认证方式：无认证 / Basic Auth
```

错误信息：
```text
粘贴完整错误，注意不要贴真实密码
```

请检查连接测试逻辑和错误处理，不要修改其他功能。
```

## 如果是界面功能错误

```text
我在使用【页面/按钮/功能名】时遇到问题。

操作步骤：
1. 打开……
2. 点击……
3. 输入……
4. 提交……

期望结果：
```text
应该发生什么
```

实际结果：
```text
实际发生什么
```

控制台错误：
```text
如果有，粘贴在这里
```

请定位问题并最小化修改。
```

## 修 Bug 的约束

每次修 Bug 都建议附加：

```text
请注意：
1. 不要大范围重写。
2. 不要顺手重构。
3. 不要增加新功能。
4. 只修复本错误。
5. 修复后给出验证步骤。
```

# AI 开发协作总说明

本目录用于指导 AI 按照 `AI_COLLABORATION_GUIDE.md` 和版本计划文件进行迭代开发。当前进入版本推进期：`releases/` 只放已完成或已发布的版本说明，`future-releases/` 放后续待实现的单版本计划。

## 使用方式

每次只交给 AI 一个版本计划文件，不要一次性要求 AI 完成多个版本。

## 核心原则

```text
1. 每次只实现一个版本。
2. 每次开始前先让 AI 阅读 AI_COLLABORATION_GUIDE.md 和本次对应的版本计划文件。
3. 明确告诉 AI 不要实现后续版本功能。
4. 修改代码前先让 AI 给出计划和文件清单。
5. 实现完成后必须说明如何启动和验证。
6. 出错时只修相关问题，不要大范围重写。
7. 版本完成后，再把对应计划整理为正式 release note。
```

## 通用提示词

```text
请先阅读 AI_COLLABORATION_GUIDE.md。

请再阅读本次对应的版本计划文件：
- 未来版本：`ai-dev-steps/future-releases/VX.Y.Z_PLAN.md`
- 已完成版本：`ai-dev-steps/releases/VX.Y.Z_RELEASES.md`

本次只实现文档中的【VX.Y.Z：XXX】。

要求：
1. 严格按照本版本的功能范围实现。
2. 不要实现后续版本的功能。
3. 修改代码前，先告诉我你准备修改哪些文件。
4. 优先保持代码简单，不要过度封装。
5. 不要引入额外技术栈，除非先说明原因并征求确认。
6. 实现完成后，告诉我：
   - 完成了哪些功能
   - 修改了哪些文件
   - 如何启动
   - 如何验证
   - 还有哪些未完成项

现在请开始。
```

## 防止 AI 跑偏的固定约束

后续每次让 AI 开发时，可以附加以下内容：

```text
请注意：
1. 不要实现本版本之外的功能。
2. 不要重构无关代码。
3. 不要为了未来版本提前设计复杂抽象。
4. 不要加入文档中没有要求的新依赖。
5. 保持代码简单，可运行优先。
6. 如果发现文档要求不明确，先问我，不要自行扩展。
7. 如果本版本已完成，不要顺手补做后续版本内容。
```

## 推荐开发节奏

```text
第 8 次：V0.3.8
```

## 每个版本完成后的检查

每个版本完成后，建议继续让 AI 执行：

```text
请根据 AI_COLLABORATION_GUIDE.md 和本版本计划，检查刚才实现是否符合要求。

请输出：
1. 已完成项
2. 未完成项
3. 是否误实现了其他版本功能
4. 是否存在 TypeScript 类型问题
5. 是否存在 Electron 安全问题
6. 如何手动验证
7. 本版本是否应该整理为正式 release note
```

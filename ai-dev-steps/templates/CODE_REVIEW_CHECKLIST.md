# 阶段完成后的代码检查清单

每完成一个阶段后，建议让 AI 根据本清单检查实现质量。

## 给 AI 的提示词

```text
请根据 ES_DESKTOP_CLIENT_PLAN.md 和当前阶段文档，检查当前实现是否符合要求。

检查范围：
1. 是否实现了本阶段要求的功能。
2. 是否误实现了后续阶段功能。
3. 是否有明显重复代码。
4. 是否有不必要的复杂抽象。
5. 是否有 Electron 安全问题。
6. 是否有 TypeScript 类型问题。
7. 是否有未处理的错误提示。
8. 是否有不必要的新依赖。

请给出：
- 已完成项
- 未完成项
- 需要修复的问题
- 是否建议进入下一阶段
- 手动验证步骤
```

## 通用检查项

```text
1. 项目是否能启动？
2. 页面是否能正常渲染？
3. 控制台是否有明显错误？
4. TypeScript 是否有类型错误？
5. 新增代码是否符合当前阶段范围？
6. 是否修改了无关文件？
7. 是否引入了文档外的新技术？
```

## Electron 安全检查

```text
1. contextIsolation 是否为 true？
2. nodeIntegration 是否为 false？
3. 渲染进程是否没有直接使用 Node.js API？
4. 是否通过 preload 暴露有限 API？
5. 是否避免把敏感信息打印到控制台？
```

## Elasticsearch 调用检查

```text
1. ES 请求是否统一在主进程执行？
2. 是否通过 IPC 调用？
3. 错误信息是否返回给前端？
4. 是否避免在日志中输出密码？
5. 是否处理连接失败���况？
```

## UI 检查

```text
1. 是否有加载状态？
2. 是否有错误提示？
3. 是否有空数据提示？
4. 表单是否有基本校验？
5. 危险操作是否有确认框？
```

## 不允许的问题

```text
1. 为当前阶段之外的功能写大量代码。
2. 大范围重构已有代码。
3. 引入未说明的新框架。
4. 把密码直接打印到日志。
5. 在渲染进程直接访问 Elasticsearch。
6. 删除已有可用功能。
```
